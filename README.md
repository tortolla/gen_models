# README

## 1. Общая структура

Каждая папка в директории models - название модели:
- `models/BLIP`
- `models/BLIP2`
-  .....


## 2. Структура папки модели

Каждая модель имеет одинаковую структуру:

- `data/`          – примеры для тестового инференса 
- `src/`           – код загрузки модели
- `inference.ipynb` – тестовый инференс модели
- `requirements.txt` – список зависимостей
- `Dockerfile`     – Docker-образ
- `docker-compose.yml` – сценарий запуска Docker-образа

---

## 3. Требования

1. Docker  
2. Docker Compose  
   - в новых версиях используется команда `docker compose`
   - в старых – `docker-compose`

---

## 4. Как запустить модель

```bash
cd models/<MODEL_NAME>
docker compose up
(если вдруг решили запустить без GPU - открываем docker-compose.yml в соответствующей папке и удаляем строчку ,, gpus: all ,,)
```bash
на портах 127.0.0.1:8888/127.0.0.1:8000 должен открыться ноутбук. (первые одна-две ячейки скачивают модель, далее идет простой пример использования)
